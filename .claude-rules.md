# Claude Rules - HaqqiAI Project

## Tech Stack Constraints

### ALWAYS USE
| Category | Technology |
|----------|-----------|
| **Framework** | React 19 with TypeScript |
| **Build Tool** | Vite with Tailwind CSS plugin |
| **Icons** | Lucide React (for components) or Material Symbols (for branding) |
| **Styling** | Tailwind CSS utility classes |
| **Routing** | State-based routing (NO external router) |
| **State** | Local useState - NO Redux/Zustand |
| **Animations** | Tailwind `animate-in` or Framer Motion `motion` |

### NEVER USE
- **External Routers**: No `react-router`, `next/router`, or similar
- **State Management**: No Redux, Zustand, Jotai, Context API for routing
- **CSS Frameworks**: No Bootstrap, Bulma, or custom CSS files
- **Component Libraries**: No Material UI, Ant Design, Chakra UI
- **Icon Libraries**: No custom SVG icons (use Lucide or Material)

## RTL (Right-to-Left) Arabic/Darija Requirement

### Global Direction
```html
<!-- ALWAYS set on root elements -->
<div dir="rtl">
<input dir="rtl" />
<html lang="ar" dir="rtl">
```

### RTL Layout Rules
1. **Icon Direction**: Chevron icons must point LEFT for navigation (clockwise)
   ```tsx
   <ChevronLeft />  // Points left (correct for RTL)
   <ChevronRight /> // Points right (correct for RTL)
   ```

2. **Flex Layouts**: Use `gap-x-reverse` or manual spacing
   - Left/right are reversed in RTL
   - Content flows from right to left

3. **Text Alignment**: Always `text-right` for Arabic content
   - `text-left` only for English URLs, numbers, or code

4. **Input Fields**: Add `dir="rtl"` for inputs with Arabic text
   - English placeholders/inputs use `dir="ltr"`

5. **Icons that Change Direction**:
   - Arrow icons point LEFT (←) when moving forward in RTL
   - Back buttons use `ArrowLeft` icon
   - Next buttons use `ArrowRight` icon

### Arabic Text Guidelines
- Use **Noto Sans Arabic** or **Noto Naskh Arabic** fonts
- Keep English text minimal (URLs, codes, technical terms)
- Darija (Moroccan Arabic) for chat interfaces
- Standard Arabic for formal documents

## Component Naming Conventions

### File Naming
- **Screens**: `PascalCaseScreen.tsx` (e.g., `ChatScreen.tsx`)
- **Components**: `PascalCase.tsx` (e.g., `TopNav.tsx`)
- **No kebab-case or snake_case**

### Component Prop Names
- `setCurrentScreen: (screen: Screen) => void` - Navigation handler
- `navigateToChat?: (prompt?: string) => void` - Chat navigation (optional)
- `isOpen` / `setIsOpen` - Modal/sidebar state
- `showMenu` / `setShowMenu` - Menu state

### State Variable Naming
- `activeTab` - Current tab in tabbed interface
- `selectedType` / `selectedCategory` - Selection state
- `isTyping` - Loading/processing state
- `expandedCard` - Accordion state
- `showModal` / `showOverlay` - Modal visibility

### CSS Class Naming (Tailwind)
- **No custom CSS classes** - use utility classes only
- **Exception**: Global styles in `index.css`:
  - `.geometric-bg`
  - `.premium-shadow`
  - `.gold-gradient-text`
  - `.nav-blur`

## RTL-Related Patterns to Follow

### Navigation Component Pattern
```tsx
// Navigation items flow right to left
<div className="flex items-center gap-2">
  <span>الГлавная</span>  {/* Content */}
  <ChevronLeft />        {/* Points right (visually) */}
</div>
```

### Search Input Pattern
```tsx
<input
  type="text"
  dir="rtl"              // Always RTL for Arabic inputs
  placeholder="ابحث..."
  className="text-right" // Align text to right
/>
```

### Modal Pattern
```tsx
// Modals use RTL by default (inherited from parent)
<div className="fixed inset-0 bg-black/40 z-50 flex items-center justify-center">
  <div className="bg-white rounded-2xl p-6 w-full max-w-md animate-in zoom-in-95">
    {/* Modal content - dir="rtl" inherited */}
  </div>
</div>
```

### Toggle Switch Pattern
```tsx
<div className={`w-[44px] h-[24px] rounded-full p-1 ${isOn ? 'bg-[#1B3A6B]' : 'bg-[#D1D5DB]'}`}>
  <div className={`w-[16px] h-[16px] bg-white rounded-full ${isOn ? '-translate-x-[20px]' : 'translate-x-0'}`} />
</div>
```

## Common UI Patterns to Implement

### Sidebar Toggle Pattern
```tsx
<button onClick={() => setIsOpen(!isOpen)}>
  {isOpen ? <ChevronRight /> : <ChevronLeft />}
</button>
```

### Search with Suggestions
```tsx
<input
  value={searchQuery}
  onChange={(e) => setSearchQuery(e.target.value)}
  placeholder="ابحث..."
/>
{searchQuery.length > 1 && (
  <div className="absolute top-full bg-white rounded-xl shadow-xl">
    {/* Search results */}
  </div>
)}
```

### Modal Overlay Pattern
```tsx
{showModal && (
  <div className="fixed inset-0 bg-black/40 z-[400] flex items-center justify-center">
    <div className="bg-white rounded-2xl w-full max-w-md animate-in fade-in">
      {/* Modal content */}
    </div>
  </div>
)}
```

### Loading/Typing Animation
```tsx
{isTyping && (
  <div className="flex items-center gap-2 animate-pulse">
    <Loader size={16} className="animate-spin" />
    <span>جاري المعالجة...</span>
  </div>
)}
```

## Accessibility Requirements

### Keyboard Navigation
- Escape key closes modals and menus
- Tab navigation works through all interactive elements
- Focus indicators visible on all interactive elements

### Screen Reader Support
- All buttons have text labels (no icon-only buttons)
- Modal titles as headings
- Status messages announced to screen readers

### Color Contrast
- Navy (#1B3A6B) on white meets WCAG AA
- Gold (#C9A84C) on dark backgrounds meets AA
- Text size: minimum 13px for body, 14px+ for interactive

## Code Quality Standards

### TypeScript
- All component props typed
- No `any` types
- Enum for screen types (via `Screen` type)

### Component Structure
- Import statements at top (lucide-react, types)
- Interface definition for props
- State variables with proper typing
- Event handlers before render
- JSX last

### Styling
- All styling via Tailwind classes
- No inline styles (except animation styles)
- Consistent spacing (p-4, p-6, gap-4, gap-6)
- Consistent border radius (rounded-xl, rounded-2xl, rounded-3xl)

## Performance Considerations

### Image Optimization
- Use `referrerPolicy="no-referrer"` for external images
- Add explicit `width` and `height` when possible

### List Rendering
- Use `key` prop for all list items
- Keep lists under 50 items for optimal performance

### Animations
- Use CSS animations for simple transitions
- Limit simultaneous animations
- Use `will-change` for animated elements

## File Organization

### New Files
- Add new screens to `src/screens/` with `Screen.tsx` suffix
- Add reusable components to `src/components/`
- Never modify `index.html` or `vite.config.ts` unless adding dependencies

### Import Order
1. React imports
2. Lucide React icons
3. Type imports (`types.ts`)
4. Local component imports

Example:
```tsx
import React, { useState } from 'react';
import { Search, Home, FileText, ChevronLeft } from 'lucide-react';
import { Screen } from '../types';
```
